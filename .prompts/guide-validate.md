# Step Guide Validation

**Purpose**: Review step guide for errors, security issues, and implementation quality.

## Instructions

Analyze the current `step-guide.md` and identify:

- **Technical Errors**: Implementation mistakes, incorrect patterns, or architectural issues
- **Security Issues**: Missing input validation, key material exposure, or security vulnerabilities  
- **Pattern Compliance**: Alignment with established patterns from `style-guide.md`
- **Go Best Practices**: Modern Go idioms, error handling, and code organization
- **Performance Considerations**: Efficiency concerns or optimization opportunities
- **Completeness**: Missing implementation details, insufficient testing, or unclear requirements

Provide specific recommendations for improving the step guide before session execution.

**Output**: Validation findings and improvement recommendations
